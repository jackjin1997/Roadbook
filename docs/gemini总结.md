### 1. 项目核心与产品定位

* **痛点切入：** 针对“JD 解析”、“简历吹水复习”以及“新概念扫盲”场景。
* **核心逻辑：** 区别于被动问答的 RAG，这是一个**主动发散与构建**的系统。它以一段非结构化文本（如 JD、技术文章）为锚点，向外扩张调研，最终收敛为结构化的学习路径。
* **MVP1 交付物：** 一份高信噪比的、支持 Obsidian 双链或原生渲染 Mermaid.js 的 Markdown 文档，主打“图文结合 + 教程导向”。

### 2. 敲定的命名与品牌感

* **核心引擎（开源/后端名）：Ariadne**
* *典故：* 希腊神话中指引忒修斯走出迷宫的线团。极客且硬核。


* **交付产物（用户端名）：路书 (Roadbook)**
* *定位：* 接地气、务实，完美契合“通关指南”和“学习路径”的工具属性。


* **Slogan：** *“输入一份 JD，Ariadne 为你生成专属的通关路书。”*

### 3. 技术架构预研 (Node.js 生态)

作为后端主导的项目，MVP1 的技术栈保持轻量和高可控性：

* **工作流编排：** 采用 **LangGraph.js**。利用其状态机特性，完美控制“节点提取 -> 联网调研 -> Markdown 聚合”的阶段性流转，甚至处理搜索结果不佳时的循环重试。
* **数据流转：** 依靠严格的 JSON Schema 约束 LLM，输出标准化数据（Skill -> Sub-skill -> Related Concepts -> Links）。
* **可视化方案：** 放弃复杂的图数据库，直接在 Markdown 中生成 `mermaid` Mindmap 代码，低成本实现脑图效果。

### 4. 首次“野生”用例测试：StreamBridge

刚才你抛出的 `StreamBridge` 是一个绝佳的真实测试用例。如果 Ariadne 引擎拿到这个词，它的内部流转应该是这样的：

1. **歧义识别：** 发现它既指代 Java Spring Cloud Stream 的消息队列桥接 API，也指代苹果最新的多模态连续视频流 AI 框架。
2. **上下文对齐：** 如果你输入的是一份后端 JD，Agent 会自动剪枝，顺着 Java/微服务/消息积压的知识树向下调研；如果是 AI 论文，则向外延展显存压缩、激活模型的知识点。
